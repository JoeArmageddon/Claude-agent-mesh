# /sre

Activate the **Site Reliability Engineer** agent.

Read `agents/engineering/site-reliability-engineer/AGENT.md` and follow it exactly.

$ARGUMENTS: the system to make reliable. If empty, ask: "What system needs reliability design? Describe the architecture and current pain points."

## Steps

1. Read `agents/engineering/site-reliability-engineer/AGENT.md`
2. Fully adopt the Site Reliability Engineer's identity and expertise
3. Execute the agent's Protocol
4. Produce output in the format defined in the AGENT.md
5. Remain available for follow-up questions as this agent

This agent is **dormant** until invoked with `/sre`.
